① Log roast 烘豆記錄開一次，記整批

Tools → Log roast · Single origin
  • 選生豆 → 填 Green in（生豆 kg）→ 秤了熟豆就填 Roasted out（失重率自動算）→ 按「＋ Add this batch」加進本批清單。
  • 連烘同一支豆：加完鍋豆子保持選中，改 kg 再按加就好。
  • 熟豆還沒秤？照樣加鍋——之後去 Coffee Stock 的 TO WEIGH 區補。
  • 烘到一半鎖屏、關 app、換裝置都不會丟（雲端暫存），打開繼續記。
  • 全部烘完按 Submit：一次寫入所有鍋、生豆一次扣掉。中途改錯／刪鍋都行，Submit 前不動庫存。
  • 烘焙曲線（Charge / First crack / Drop / 時間）每鍋選填，收在 Roast log 摺疊區。

② Coffee Stock 熟豆庫存倉庫盤點

Home → Coffee Stock

血條＝可賣庫存，顏色是新鮮度：黃 aging（養豆中）→ 綠 ready → 紅 fading。三種批次池分開管：

可賣池正常庫存，出貨自動從最舊批次扣（FIFO）
To weigh 待秤烘了沒秤熟豆——點行填 kg 即入庫
Downgraded 降級不算可賣、訂單吃不到；拼配烘焙優先消耗
  • 0kg 的單品不顯示——例外：有待秤或降級批次的照樣現身。
  • 膠囊三態：LIVE 上架且鎖了＝客人買得到｜NO LOCK 掛在 Square 但沒鎖＝客人看不到，去 QC 鎖｜OFF SHELF 手動下架。
  • 「−」扣豆記原因（Defect／Transfer／Other）；打錯數字用 Fix kg 修正，不會污染瑕疵帳。

③ QC 杯測與判定鎖＝店面閘門

Home → QC
  • QC 名單只來自 Coffee Stock——有庫存的批次才會出現在待測清單。
  • Cup：記風味詞、雷達五維、comment。
  • 判定用滑的：右滑＝Pass ＋ 自動鎖風味 🔒；左滑＝Downgrade（批次回 Coffee Stock 帶紅標，等拼配吃掉）。
  • 鎖的規則：同一支豆（名字＋處理法）只有一把鎖＝正在賣的版本。不同處理法各自有鎖、可以同時在賣（例：Alo Village 的 White Honey 和 Cold Fermentation 是兩支商品）。
  • 烘出更好的批次 → Pass 它會換鎖（有確認提示）。沒鎖的舊紀錄＝次要批次，之後可刪。

④ Publish 上架與推廣一鍵到店面

Home → Publish
  • 清單只列鎖了風味的豆（跟 Coffee Stock 同源、豆名旁標處理法）。
  • List on Square + Ratio shop：建立／更新 Square 商品＋?shop 店面。自動歸類 Blend ／ Single Origin，並有防護罩不會抓錯店內 POS 品項。
  • Take off shelf：豆子從店面完全消失（不掛售罄卡）；Put back on sale：回架。
  • 同面板：Email 客人新豆通知、FB/IG 發文、IG 素材卡。

⑤ Coffee Info 豆子檔案室旁支 · 純看資訊

Tools → Coffee Info
  • In stock：鎖了＋有庫存＝正在賣的版本（含上架膠囊、價格提醒）。
  • Pending：有庫存還沒鎖＝待辦——去 QC 鎖來上架。
  • History：鎖了但賣完＝存檔，查風味、雷達、履歷。
  • 排序：烘豆日舊在上（先賣先清）；點豆開詳情（可 Send back to QC 重審）。上架動作統一走 Publish。

鐵律

  1. 豆子雙鍵：名字＋處理法。同名不同處理法＝不同商品，永遠分開看。
  2. 風味鎖＝店面閘門：沒鎖的豆客人永遠看不到，掛著也沒用（NO LOCK 就是在提醒）。
  3. 降級豆不隱形：留在 Coffee Stock 帶紅標，只給拼配吃，訂單永遠吃不到。
  4. Off shelf＝從店面消失：下架的豆不再掛售罄卡片，回架自動出現。
  5. 出貨自動扣庫存：FIFO 最舊批次先出，不用手動記。
